# NovaNest AI

> Your AI career growth OS — build ATS-optimized resumes, practice role-specific interviews, generate cover letters, and track live industry insights, all in one polished workspace.

NovaNest AI is a full-stack SaaS application built with **Next.js 15 (App Router)**, **React 19**, **Tailwind CSS + shadcn/ui**, **Prisma + PostgreSQL**, **Clerk** authentication, **Google Gemini** AI, and **Inngest** background jobs. It is designed to feel like a real, commercially-viable startup product — premium dark-first design system, robust server-action architecture, rate-limited AI pipelines, and production deployment readiness.

---

## ✨ Features

| Module | What it does |
| --- | --- |
| **NovaScore** | A composite 0–100 career-readiness score across resume, interview, cover-letter, application pipeline, and market-fit pillars — with daily streaks & XP that grow as you use the app. |
| **Industry Insights** | AI-analyzed salary ranges, growth rate, demand level, top skills, key trends, and a personalized **skill-gap** analysis (your skills vs. market demand). Refreshed weekly via Inngest cron. |
| **Resume Builder** | Dynamic, ATS-friendly resumes with a live Markdown editor, AI entry improvement, one-click PDF export, and autosave. |
| **Interview Prep** | Role-specific AI quiz generation, progress tracking, performance trend analytics, and AI improvement tips after each attempt. |
| **Voice Mock Interview** | A spoken mock interview using the browser's Web Speech API — the AI asks one question at a time, you answer out loud, and the full transcript is scored by Gemini. |
| **Cover Letters** | AI cover-letter generation from a job description, with search, sort, and filtered management. |
| **Application Tracker** | A drag-and-drop Kanban pipeline (Saved → Offer) with resume↔JD **ATS matching** — score, matched/missing keywords, and concrete edits. |
| **AI Career Tools** | Achievement rewriter (STAR/XYZ), 8-week skill roadmap generator, cold-outreach message writer (LinkedIn/email), and a job-fit scorer. |
| **Weekly Brief** | A personalized Monday career digest (market pulse, skill to watch, practice question, resume tip, action item), generated by an Inngest cron and shown on the dashboard. |
| **Auth & Onboarding** | Clerk-backed sign-in/up, profile onboarding with industry/skills selection, and protected routes that gate on onboarding completion. |

## 🧱 Tech stack

- **Framework:** Next.js 15 (App Router, RSC, Server Actions, Turbopack dev)
- **UI:** Tailwind CSS, shadcn/ui, Radix primitives, Framer Motion, Recharts, Lucide icons
- **Auth:** Clerk (with NovaNest-themed appearance)
- **Database:** PostgreSQL via Prisma ORM
- **AI:** Google Gemini behind a shared client with a **model fallback chain** (primary `gemini-3.5-flash`; override via `GEMINI_MODEL`) + robust JSON parser
- **Background jobs:** Inngest (weekly insight refresh cron)
- **Forms/validation:** React Hook Form + Zod
- **Notifications:** Sonner

## 🏗️ Architecture

```
┌──────────────────────────────────────────────────────────────┐
│  Browser (RSC + client islands, framer-motion, next-themes)  │
└───────────────────────────────┬──────────────────────────────┘
                                │  Server Actions (RPC over POST)
┌───────────────────────────────▼──────────────────────────────┐
│  actions/  ── requireUser() ── zod boundary validation ──┐   │
│  (resume, interview, cover-letter, dashboard, user)      │   │
└──────────────────────────────────────────────────────────┼───┘
                          │                                  │
        ┌─────────────────┼──────────────┐                   │
        ▼                 ▼              ▼                   ▼
  lib/ai/gemini.js   lib/rate-limit   lib/prisma     Inngest cron
  + lib/ai/prompts    (token bucket)  (Postgres)     (weekly refresh)
        │
        ▼
  Google Gemini API
```

### Key design decisions

- **Single AI client** (`lib/ai/gemini.js`): replaces five duplicated `new GoogleGenerativeAI(...)` instantiations with one shared client and a robust `parseJSONResponse` (balanced-brace extraction) so fenced or prose-wrapped model output never crashes a page.
- **Centralized prompts** (`lib/ai/prompts.js`): the on-demand insight generator and the weekly Inngest cron share the exact same prompt contract, keeping insights consistent.
- **Auth boundary** (`lib/auth.js`): `requireUser()` replaces the copy-pasted `auth() → db.user.findUnique` block in every action and throws typed `UnauthorizedError` / `UserNotFoundError`.
- **Rate limiting** (`lib/rate-limit.js`): in-memory token bucket guards every paid Gemini call (quiz, cover-letter, improve-entry, insight generation) against abuse.
- **Typed errors** (`lib/errors.js`): `AppError` hierarchy + `toPublicMessage` ensures toasts never leak DB/AI internals.
- **Onboarding guard** (`lib/onboarding.js`): signed-in users without a completed profile are redirected from every protected page.

## 🚀 Getting started

### Prerequisites
- Node.js **>= 20** (see `.nvmrc`)
- A PostgreSQL database (e.g. [Neon](https://neon.tech))
- A Clerk application
- A Google Gemini API key

### Install
```bash
npm install
```

### Configure
```bash
cp .env.example .env.local
# Fill in DATABASE_URL, Clerk keys, GEMINI_API_KEY, NEXT_PUBLIC_APP_URL
```

### Database
```bash
npx prisma migrate deploy   # apply existing migrations
# or, for local dev:
npx prisma migrate dev
```

### Run
```bash
npm run dev      # http://localhost:3000 (Turbopack)
```

### Scripts
| Script | Description |
| --- | --- |
| `npm run dev` | Start the dev server |
| `npm run build` | Production build |
| `npm run start` | Run the production build |
| `npm run lint` | ESLint |

## ☁️ Deployment (Vercel + Neon)

1. Push the repo to GitHub.
2. Import the project into [Vercel](https://vercel.com) — framework preset **Next.js** is auto-detected.
3. Add all variables from `.env.example` in **Project → Settings → Environment Variables**.
4. Create a PostgreSQL DB on [Neon](https://neon.tech), set `DATABASE_URL`, run `npx prisma migrate deploy` (Vercel runs `postinstall: prisma generate` automatically).
5. For the weekly insight refresh, deploy Inngest: point the Inngest dashboard at `https://<your-domain>/api/inngest`, or run the [Inngest Dev Server](https://www.inngest.com/docs/local-development) locally with `npx inngest-cli@latest dev`.
6. Deploy. 🎉

See [`docs/DEPLOYMENT.md`](docs/DEPLOYMENT.md) for the full checklist.

## 🔒 Security & reliability

- Clerk-managed auth; protected routes via `middleware.js` + per-page onboarding guards.
- Server-action input validation with Zod at the boundary.
- Rate-limited AI calls (token bucket per user).
- Robust AI JSON parsing — malformed model output degrades gracefully.
- Security headers on every response (`X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Permissions-Policy`, HSTS).
- Typed error layer prevents internal leakage in toasts.
- Global `error.jsx` boundary + `loading.jsx` states across routes.
